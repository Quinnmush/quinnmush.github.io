## 欢迎

这是博客的第一篇文章，使用 **Markdown** 编写。

### 关于本站

这个博客使用 Astro 构建，内容以 Markdown 文件形式管理，通过 GitHub Pages 自动部署。没有数据库，没有 CMS，只有纯粹的静态文件。Astro 的岛屿架构让我们可以按需加载 JavaScript。

## 技术选型

### 静态站点生成器

Astro 5 是目前最新的版本，它提供了内容集合（Content Collections）功能，让我们可以方便地管理 Markdown 文章。

### 样式方案

本站使用纯 CSS 配合 CSS 自定义属性（设计 Token），没有引入任何 CSS 框架。设计 Token 定义在 `variables.css` 中，包括颜色、字体、间距和布局变量。亮暗模式通过 `data-theme` 属性切换。

### 搜索功能

使用 MiniSearch 实现客户端全文搜索。构建时通过 strip-markdown 工具将文章正文转为纯文本，生成静态 JSON 索引。浏览器端建立索引，支持 CJK 分词、模糊匹配和前缀搜索。

### 代码高亮

使用 Astro 内置的 Shiki 进行代码语法高亮。双主题：日间 github-light，夜间 github-dark。

### 评论系统

集成了 giscus，基于 GitHub Discussions。评论主题跟随站点亮暗模式同步切换。

## 设计原则

### 内容优先

博客设计的核心原则是内容优先。单栏布局、最大宽度 720px、无侧边栏。

### 色彩系统

日间模式：暖白背景 `#faf6f0`，深色文字 `#2c2c2c`，强调色 `#c0552d`。夜间模式：深灰背景 `#1a1a1e`，浅色文字 `#e4e4e7`，强调色 `#e88d6a`。

### 字体

正文使用 Inter（无衬线字体），行高 1.75。代码使用 JetBrains Mono（等宽字体）。

## 部署流程

### GitHub Actions

每次推送代码到 main 分支时，GitHub Actions 自动构建并部署到 GitHub Pages。

### 发布策略

写入流程：本地写 `.md` 文件 → `npm run dev` 本地预览 → 发布时改 `draft: false` → Git push 触发自动部署。

## 测试内容

段落一：静态站点生成器在现代 Web 开发中越来越受欢迎。它们结合了静态网站的速度优势和动态网站的内容管理灵活性。

段落二：CSS 自定义属性（设计 Token）为样式管理提供了强大的基础。通过定义一套统一的颜色、间距、字体变量，可以确保整个站点视觉风格的一致性。

段落三：客户端搜索是一个有趣的技术挑战。MiniSearch 作为轻量级全文搜索库，在浏览器端建立倒排索引，支持模糊匹配和前缀搜索。

段落四：亮暗模式切换已经成为现代网站的标配功能。实现时需要注意三层降级策略：用户手动选择、系统偏好、默认日间。

段落五：阅读进度指示器是小而美的交互细节。桌面端使用 TOC 侧栏的竖向进度条，移动端使用导航栏底部的横向进度条。
